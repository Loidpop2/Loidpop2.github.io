Workspace - Whiteboard

    Hey, reader! Currently, I'm testing out a system for a sort of white board that you can use on the site! Maybe you want to jot down some ideas? Or just brainstorm for a school project? Check this out!

    Now, from the looks of it, you might be thinking— I don't know how to code! Well, that's ok. I'll teach you! Okay, maybe you weren't thinking that— and now you're probably wondering what coding has to do with a whiteboard. I don't blame you, but trust me when I say this— it's worth it to learn how to use this tool.

    Okay, but this blog is just reading so far. Where's the actual tool? If you want to skip the docs, or have used it before, the tool is right here. Otherwise, I highly reccomend you read this blog post first.

    Now, first thing's first— where do I even type the code? Well, you'll want to go to the whiteboard tool in a new tab to actually use this, but you should keep this tab open so you can read up. Now, let's begin!

    Ok, let's learn about the functions you can use to illustrate your ideas. The first one is "note". Now, similar to how it sounds, it basically puts a sticky note on your whiteboard. How do you use it? That's where syntax comes in. Now if you already know what syntax is, you can probably just skip forward a bit– if not, though, let me teach you.

    Alright, what's syntax exactly? It's basically the format of how you run code. Okay, wait, but didn't I say this was just a whiteboard system? Again, it is, but it's not like you're coding it from the bottom up. Here's an example of a whiteboard that you can make with just a few lines of code:

    So... how do you go about making that? Again, syntax. It's better if I give you an example, yeah?

note("Hello World!",5,5,10,10,rgb(239, 255, 19));

    Ok, maybe that's a little misleading. Those commas divide different variables- or parameters in this case- that get sent to the "note" function, which, in turn, tells it what to do. Here's what each of the variables do:

note("text",posx,posy,width,height,rgb(red,green,blue));

    Alright, that's more clear. But in the example, there's images and fonts and all that... how do you use those? Well, syntax doesn't just imply that one "note" function- it applies to every function. For example, the "image" function has it's own syntax, like so:

image("https://d22l0cevb7ryns.cloudfront.net/wp-content/uploads/2020/07/Messier-782-28-20True-Color.jpg",5,20,10);

    Again, I'll cut to the chase- here's what each parameter means:

image("url",posx,posy,resolution);

    Okay, wait, what? What's the URL? All a URL does is just point to an online resource, just like how "https://google.com" brings you to Google. But... what URL do you put? How do you direct a URL to an image? My tactic is to go to Google, find an image I like, right click it, click "Copy Link Address" (this puts the URL onto your clipboard), then paste it (CTRL + V typically) where I need the URL to go. Make sense? Perfect.

    Well, where do you even go to program your whiteboard? Right here, or you can click the big button at the top of this blog for easier access.

    Before you go, though, you're probably going to want to know all of the functions. Here's a quick table of all of them, along with their syntax. Now, go create!

Function Syntax
note note(text, posx, posy, width, height, rgb(red,green,blue));
image image(url, posx, posy, scale);
text text(text, posx, posy, fontsize, rgb(red,green,blue));
font font(fontID)
