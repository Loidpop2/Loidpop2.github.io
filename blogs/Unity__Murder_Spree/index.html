Unity - Murder Spree

    Hello reader! Lately, I've been working on a project with Unity. Have you played games like Risk of Rain? It's a game like that, except it will have an estimated 790 trillion items! That's right— 790 trillion. Currently it has about 420 million, but because I used item combinations, I was able to get the size in just a few megabytes!

    Now, you're probably thinking— how on earth did I fit a few trillion items into a few megabytes? Well, I'll explain. Dotnet is what Unity runs off of— hence the C#— except... Dotnet is a coding language with so many libraries. I set up a system to compile code during runtime— meaning turn some variables into the code the computer can actually run and execute— and stored combinations of items in a JSON.
